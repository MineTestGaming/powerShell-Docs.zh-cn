## <a name="dependency-on-native-libraries"></a><span data-ttu-id="aa2ff-183">本机库的依赖项</span><span class="sxs-lookup"><span data-stu-id="aa2ff-183">Dependency on Native Libraries</span></span>

<span data-ttu-id="aa2ff-184">旨在跨不同操作系统或处理器体系结构使用的模块可能依赖于托管库，而托管库本身又依赖于一些本机库。</span><span class="sxs-lookup"><span data-stu-id="aa2ff-184">Modules intended for use across different operating systems or processor architectures may depend on a managed library that itself depends on some native libraries.</span></span>

<span data-ttu-id="aa2ff-185">在使用 PowerShell 7 之前，必须使用自定义代码来加载适当的本机 dll，以便托管库能够正确找到它。</span><span class="sxs-lookup"><span data-stu-id="aa2ff-185">Prior to PowerShell 7, one would have to have custom code to load the appropriate native dll so that the managed library can find it correctly.</span></span>

<span data-ttu-id="aa2ff-186">借助 PowerShell 7，可以在托管库位置（在 [.NET RID 目录][]表示法的子集后面）的子文件夹中搜索要加载的本机二进制文件。</span><span class="sxs-lookup"><span data-stu-id="aa2ff-186">With PowerShell 7, native binaries to load are searched in sub-folders within the managed library's location following a subset of the [.NET RID Catalog][] notation.</span></span>

                |--- 'win-arm64' folder
