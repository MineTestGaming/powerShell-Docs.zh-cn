> [!NOTE]
> <span data-ttu-id="c1e2d-138">尽管 `Set-ItemProperty` 具有 Filter  、Include  和 Exclude  参数，但它们无法用于按属性名称进行筛选。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-138">Although `Set-ItemProperty` has **Filter** , **Include** , and **Exclude** parameters, they cannot be used to filter by property name.</span></span> <span data-ttu-id="c1e2d-139">这些参数引用注册表项（即项路径），而不引用注册表条目（即项属性）。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-139">These parameters refer to registry keys—which are item paths—and not registry entries—which are item properties.</span></span>

<span data-ttu-id="c1e2d-140">另一种方法是使用 Reg.exe 命令行工具。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-140">Another option is to use the Reg.exe command line tool.</span></span> <span data-ttu-id="c1e2d-141">有关 reg.exe 的帮助，请键入 **reg.exe /?**</span><span class="sxs-lookup"><span data-stu-id="c1e2d-141">For help with reg.exe, type **reg.exe /?**</span></span>
<span data-ttu-id="c1e2d-142">。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-142">at a command prompt.</span></span>

<span data-ttu-id="c1e2d-143">下面的示例通过删除在上面的示例添加的路径来更改 Path  条目。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-143">The following example changes the **Path** entry by removing the path added in the example above.</span></span>
<span data-ttu-id="c1e2d-144">`Get-ItemProperty` 仍可用于检索当前值，以避免必须解析从 `reg query` 返回的字符串。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-144">`Get-ItemProperty` is still used to retrieve the current value to avoid having to parse the string returned from `reg query`.</span></span> <span data-ttu-id="c1e2d-145">SubString  和 LastIndexOf  方法用于检索添加到 Path  条目的最后一个路径。</span><span class="sxs-lookup"><span data-stu-id="c1e2d-145">The **SubString** and **LastIndexOf** methods are used to retrieve the last path added to the **Path** entry.</span></span>


```Output
The operation completed successfully.
```
